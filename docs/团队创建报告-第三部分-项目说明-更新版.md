## 1、项目类型

命题项目，融合创新拓展。项目来源为生产实习AI大模型方向选题“教育助手AI Agent”，结合企业入职培训实际场景进行需求分析与全栈实现。项目在10天生产实习周期内按Scrum方式推进，交付了**后端（FastAPI + LangChain + ChromaDB）**、**Web前端（Vue 3 + Naive UI）**与**Android客户端（Kotlin + Jetpack Compose）**三端可运行产品，涵盖服务端、浏览器端和移动端完整技术栈。

## 2、项目名称

**OnboardAgent（EduAssistant）—— 基于RAG的企业新员工入职培训智能助手**

## 3、开发技术平台

### ■ Web前端

Vue 3 + Vite + Naive UI + Pinia + Vue Router + Axios，构建登录/注册、对话界面、对话历史、文件上传与管理、AI记忆设置、Markdown渲染和WebSocket流式问答体验。前端为纯JavaScript（无TypeScript），通过Vite开发服务器代理转发 `/api` 与 `/ws` 请求至后端。

### ■ Android客户端

Kotlin + Jetpack Compose + Material 3（Catppuccin 主题），采用 MVVM + Repository + 手动DI架构（不使用Hilt，因AGP 9.x兼容性限制）。网络层使用 Retrofit 2.11 + OkHttp 4.12，本地存储使用 EncryptedSharedPreferences（Token安全存储）与 SharedPreferences（服务器地址持久化）。完整对接后端全部API端点，支持OkHttp WebSocket流式对话（自动重连，指数退避，最多5次），Markdown渲染（mikepenz/compose-markdown），Coil图片加载。功能覆盖：登录/注册、对话管理、文件上传/删除/摘要/知识提取、AI记忆画像管理、个人资料编辑。

### ■ 后端

FastAPI + SQLAlchemy 2.0 + SQLite + Pydantic v2，提供用户认证（JWT + bcrypt）、文件管理、对话管理、工具调用（摘要/知识提取）、AI记忆管理和Agent对话接口。通过REST API处理普通请求，通过WebSocket（`/ws/chat/{user_id}`）实现流式对话，消息类型包括 Meta / Thinking / Token / Done / Error。数据隔离采用 ChromaDB per-user collection + 数据库 user_id 外键过滤双层机制。

### ■ AI与知识库

嵌入模型使用 **BAAI/bge-small-zh-v1.5**（HuggingFaceEmbeddings，strict offline 模式，`TRANSFORMERS_OFFLINE=1`），向量库使用 **ChromaDB**（per-user collection，确定性chunk ID `user_{id}_{doc_id}_{chunk_idx}` 保证幂等）。LLM网关支持 **DeepSeek**（deepseek-chat）与 **SiliconFlow**（Qwen/Qwen3.5-397B）两种在线模型，无有效API Key时自动降级为 **LocalDemoLLM** 离线演示模式（规则引擎，覆盖摘要/知识提取/问答三种场景），确保注册→上传→RAG问答完整流程可在无网络环境跑通。

RAG全链路：PDF/Word/TXT/Markdown文档解析（PyPDF2 + pdfplumber + python-docx）→ 文本分块（800字符chunk，200字符overlap，优先段落/句子边界）→ BGE向量化 → ChromaDB语义检索（Top-4，支持文档范围过滤）→ LLM生成带 `[来源N]` 引用的流式回答 → 在线评估（检索质量、归因覆盖度、引文有效性、幻觉风险）。

### ■ 部署与工具链

**开发阶段：** Web前端运行在Vite开发服务器（:5173），Android客户端通过LAN IP（默认 `192.168.100.134:8000`）或模拟器地址（`10.0.2.2:8000`）直连后端，后端运行在Uvicorn/FastAPI（:8000）。提供一键启动脚本（`start.sh` / `start.bat` / `start.py`），自动创建venv、安装Python与Node依赖、复制 `.env.example`。系统支持**完全离线演示**：嵌入模型本地运行（`local_files_only=True`），LLM自动降级为规则引擎，无需任何外部API依赖。

**交付阶段：** 可将Web前端构建为静态资源由Nginx托管，后端通过 `--host 0.0.0.0` 对外暴露，Android端构建Release APK分发。

## 5、项目简介

OnboardAgent是一款面向**企业新员工入职培训场景**的RAG智能问答助手。系统围绕“上传企业制度资料 → 自动向量化 → 智能问答 → 可追溯来源”的核心链路，帮助新员工快速理解公司规章制度、人事流程、信息安全规范、报销政策和岗位培训材料，降低HR重复答疑成本，提升入职培训效率与知识可及性。

区别于通用聊天机器人，OnboardAgent强调**三个核心能力**：

1. **答案来源可追溯** —— 每条回答标注引用chunk编号和来源文档名称，支持逐条验证；
2. **全链路离线可运行** —— 嵌入模型本地部署（BGE-small），LLM自动降级为规则引擎，无需外部API即可完成从注册到RAG问答的完整演示流程；
3. **双端覆盖** —— 提供Web端（浏览器访问）与Android原生客户端两种使用方式，适配桌面办公与移动查询不同场景需求。

系统在技术实现上采用**单Agent架构（edu_agent）**，核心功能模块包括：

| 模块 | 说明 |
|------|------|
| 文件上传即时向量化 | 上传即检索，无需单独处理步骤 |
| RAG流式问答 | WebSocket推送，支持Agent思考过程可视化 |
| 文档摘要生成 | 制度速览，支持短/中/长三种长度 |
| 知识卡片自动提取 | JSON结构化输出，自动分为入职流程/人事制度/财务报销/信息安全/合规要求/岗位培训六类 |
| AI用户记忆系统 | 启发式规则追踪用户常问主题、回答风格偏好与沟通语气，自动注入LLM Prompt实现个性化 |
| 对话历史管理 | 最近3轮6条消息作为上下文窗口 |
| 在线回答质量评估 | 检索质量、归因覆盖度、引文有效性、幻觉风险四项在线指标，前端可视化展示 |

**Android客户端**完整复刻Web端全部功能，采用Material 3 Catppuccin（Latte/Mocha）配色方案，支持浅色/深色模式自动切换。聊天界面通过OkHttp WebSocket实现流式token实时渲染与Agent思考步骤展示，完成后自动切换为Markdown富文本格式（表格、代码块、引用等）。消息气泡支持长按复制，助手回答下方提供可折叠的思考过程面板、来源引用面板和评估指标面板。知识库管理、AI记忆设置和个人资料编辑以 ModalBottomSheet 方式集成在主界面中，6个预设问题引导新用户快速上手。

项目在10天生产实习周期内按Scrum方式完成**两个Sprint迭代**：

- **Sprint 1（5天）：** 注册登录、文件上传与自动向量化、RAG流式问答、WebSocket实时通信、ChatUI核心界面、对话历史管理
- **Sprint 2（5天）：** AI记忆系统、文档摘要与知识提取工具、在线评估模块与前端可视化、Android客户端全功能开发、UI全面美化（Catppuccin主题）、离线演示模式完善
